Function

The application manager is the main window of the PDA. It allows to start-up applications and offers a kind of start-menu. Any application that a user should be able to access should also be available through the application manager. Thus any application has to provide at least a name that is displayed as label and optional an icon. It should also be possible to sort the aplications into different categories so that finding them is easier. This could be done in a tab-like fashion like it is shown in the above picture.

For certain application it is necessary that they can only be started once, for example the address application. The address application itself should handle anything that a user could want from multiple instances of the application. On the old Psions for example I used to have several address applications running for different address databases: Home, Work, Private, etc. The application itself could handle this by offering proper categories.

Why no status-bar?

Personally I do not like to waste screen space. The status bar is mostly static and only seldomly used. Usually we will get a second bar from the application, i.e. a menu-bar. What I would like to see is a flexible menu-bar instead of a status-bar. Applications could register menus in it and it could also display system global items like time/date.

First proposal

Robert Mibus (mibus@bigpond.com) already programmed such an application and posted the following to the GPE mailinglist:

As there was an apparent lack of PalmOS/WinCE style program launchers, I wrote my own. the 'fmenu' executable can go anywhere, fmenu.glade wants to go in /usr/share/. It reads all of the files in /usr/lib/fmenu/ and /mnt/ramfs/fmenu/, the format is very simple. (There are a few examples in the tarball).

Each file has four lines.
line 1 is the proper, displayed name (eg. Dillo or Web Browser)
line 2 is the executable name (eg dillo)
line 3 is the icon name (eg /usr/share/pixmaps/dillo.png)
line 4 is the group name.

There is a clock in the top-left corner - it is actually a menu with 'refresh' and 'quit' (hitting the 'X' has been disabled because I don't want to accidentally kill it off). In the middle is an APM monitor, the top-right has the group selector.

http://users.bigpond.com/mibus/fmenu.tar.gz

Please, bash it around - iPAQ binaries included.

The only problem I know of is that when you press the group drop-down, the list (in the background) often changes to "All"; this appears to be a Gtk problem AFAICT. It doesn't affect the usability though.

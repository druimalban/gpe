Function

This is very obvious. The calendar stores and manages scheduled events. Special functionality included is support for

  • Entries whithout due-time, just a note for a particular day.
  • Reoccuring events with options for daily, workdays, weekends, monthly, yearly (birthdays or other anniversarys)
  • Beaming entries via IrDA IrOBEX in vCalendar format.
  • Different view modes to give the user a good overview.

Data to store

Here a short start of what data has to be stored: 

char[] description;
datetime starttime;
datetime endtime;
bool alarm;
datetime alarm_before_starttime; /* amount of time to ring alarm before
actual starttime */
repeattype repeat;
unsigned int flags; /* some misc. flags like enable/disable item, etc. */

enum repeattype = once, daily, weekly, monthly, yearly, workdays, weekends;
